# Top frontend rewrite

Стартовый фронтовый каркас для миграции старого PHP-сервиса `top` на:

- React
- Redux Toolkit
- styled-components
- Node.js backend API

## Что уже перенесено в архитектуру

По старому репозиторию видно минимум три главных пользовательских экрана:

1. `index.php` → **Комиссии**
2. `mo.php` → **Долг 2-го уровня**
3. `rating.php` → **Рейтинг МО**

Также сохранена логика общего layout и навигации, которая повторяется между страницами старого сервиса. В исходном проекте эти разделы действительно присутствуют как отдельные PHP-страницы и связаны общей навигацией. Это видно в структуре репозитория и сыром содержимом файлов `index.php`, `mo.php` и `rating.php`. citeturn147727view0turn152273view1turn892182view4turn892182view3

## Почему такая структура

Старый фронт рендерится на стороне PHP и визуально уже содержит:

- верхнюю навигацию;
- summary-блоки по региону;
- фильтры региона/квартала/периода;
- большие таблицы с данными;
- карточку организации / ИНН в модальном окне;
- административные и служебные разделы.

Это следует из содержимого старых страниц. Например, `index.php` содержит сводную информацию по региону, поиск, таблицу плательщиков и форму по комиссии; `mo.php` содержит summary по региону, фильтры и две таблицы; `rating.php` содержит сводные показатели, рейтинг и топы муниципалитетов. citeturn152273view1turn892182view4turn892182view3

## Предлагаемая миграция фронта

### 1. Layout

- `AppShell` — новый общий каркас вместо повторяемого PHP-header/footer.
- React Router — маршруты `/commissions`, `/mo`, `/rating`.

### 2. State management

Отдельные slices:

- `commissionsSlice`
- `moSlice`
- `ratingSlice`
- `authSlice`

Так проще сохранять старую бизнес-логику, но убрать смешение HTML + jQuery + PHP в одном файле.

### 3. UI primitives

Общие переиспользуемые элементы:

- `PageHeader`
- `SummaryGrid`
- `FilterBar`
- `DataTable`

### 4. API слой

Сейчас добавлен `axios` client с базовой настройкой `x-user-key`. Это задел под будущую замену старого `api.php?q=...` на REST/JSON endpoints в Node.js.

## Что еще не перенесено

Пока это **не полный клон старого интерфейса**, а правильный старт для миграции:

- нет модалки карточки ИНН;
- нет настоящих запросов к backend;
- нет пагинации "Загрузить еще";
- нет авторизации/логина;
- нет админки;
- нет загрузки файлов протоколов;
- нет экспорта в Excel;
- нет перенесенных legacy endpoint mapping из `api.php`.

## Следующий шаг

Лучший следующий шаг — **добить экран `Комиссии` до полноценного состояния**, потому что именно он содержит самую насыщенную бизнес-логику:

- фильтрация;
- список плательщиков;
- карточка организации;
- редактирование статуса комиссии;
- сохранение файла протокола;
- история изменений.

После этого можно перейти к Node.js API и уже на его стороне обновить старые SQL-запросы, в том числе проблемные места вроде `ONLY_FULL_GROUP_BY`.
